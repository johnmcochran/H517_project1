Design Process:

Initial approach:

I modified my visualizations with the following hierarchy in mind:

1.      The visualizations can easily be seen when the chrome browser is set to 100% zoom.

2.      Features within the visualization can each be easily distinguished from one another via color and size

3.      Colors used will be discernable by those with color blindness.

In my initial design phase, I considered how the visualizations would be oriented with respect to one another as well as the details of specific features within the visualizations, such as size of circles for the pumps and deaths for the map and whether to use a bar chart, scatterplot, line chart, or some combination for the graph showing number of deaths each day.

Rationalization of Design Choices:

I ultimately decided to keep the visualizations side by side instead of one on top of the other. Because the key feature of these visualizations is interactivity, I did not want users to need to scroll up and down to see each visual. Further, it is a necessity that both visuals be on the user's screen as they are interacting with them.

For the map, I chose to display the pumps as the largest circles, colored blue to be associated with water. One of the key purposes of the interaction is to be able to see how the location of deaths are related to the location of the pumps. Since there are so few pumps in comparison to deaths, it made the most since to have the pumps be the largest circles on the map so that they pop out for the user. For deaths, I chose to have the circles be just small enough that you individual deaths are easily discernable when there are multiple deaths in one area, but not so small that they cannot be seen by the user. The value of the circles representing deaths were adjusted to reflect gender; grey circles for female deaths, black circles for male deaths.

For the timeline graph showing number of deaths by day, I settled on using a combination of a scatterplot and line graph, where the line graph allows for an easy visual of the trend of deaths by day and the scatterplot circles allow the user to hover their mouse over the circle to color the death dots on the map red for all deaths that occurred up to that day. When the user hovers their mouse over a timeline circle, the circle turns red and becomes larger so that the user knows they have successfully interacted with the visualization.

Using the Visualizations to Uncover Trends

While looking at the map without the timeline interaction, it is difficult to see the source of the deaths from the cholera outbreak. Using the interactive feature, the deaths appear to be quite random. However, by the end of the first week of September, we are finally able to see a cluster of points suggesting a center point near one of the pumps. Hovering our mouse over the final day paints a much clearer picture that these deaths were related to a single pump. All of the deaths turn red, and there is a single blue pump in the middle.
